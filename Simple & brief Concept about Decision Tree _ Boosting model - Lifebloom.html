Simple & brief Concept about Decision Tree ~ Boosting model

일반적인 상황에서…

의사결정나무 : DT, 랜덤 포레스트 : RF, 그래디언트 부스팅 모델 : GBM, XGB, LightGBM

 

모델의 정확성(high->low) : XGB, LightGBM > GBM > RF > DT

학습 시간(fast->slow) : DT > LightGBM > XGB > GBM > RF

변수 해석 용이성(easy->hard) : DT > RF > LightGBM, XGB, GBM

-과적합(Overfitting)이 발생할 가능성이 높음
-단일 트리이기 때문에 데이터의 변화에 불안정하게 반응
랜덤포레스트
(Random Forest)		-Bagging 방식을 사용하여 모델의 과적합을 줄여서 안정성과 정확성 향상-학습에 필요한 시간이 의사결정나무에 비해 오래 걸림
-의사결정나무에 비해 각 변수들의 효과를 알기 힘듦

-사용되는 parameter가 많기 때문에 조절에 시간과 경험이 필요

-GBM에 비해 연산속도가 빠름		-변수가 모델에 미친 영향을 정확하게 아는 것이 힘듦

SUMMARY

DT – (variance가 너무 커서 정확하지 않아..) -> RF – (정확성은 올랐는데 편향[Bias] 때문에 정확성이 떨어져..)-> GBM -(학습시간이 너무 오래걸리고 과적합을 더 줄여야해)-> XGB ->(학습시간이 오래걸리고 메모리 사용도 비효율적이야..)-> LightGBM

 

 

1. Decision Tree

 

의사결정나무는 데이터 안에 존재하는 패턴을 가지고 예측 가능한 조합으로 나무의 형태로 나타낸 것.

학습된 데이터의 규칙성을 가지고 스무고개 놀이와 유사한 방식으로 새로운 데이터를 분류하는 방식

 

  • question 1) 어떤 기준을 가지고 분류를 하는가?(스무고개 질문을 어떻게 정하지?)

불확실성(Uncertainty)이 가장 감소하는 방향으로 기준을 정함

  • question 2) 불확실성이 무엇인가?

판단이나 의사결정에 필요한 정보의 부족을 불확실성이라 하며 불확실성이 감소하는 것을 정보 이론에서는 정보획득(Information Gain)이라 표현

  • question 3) 여러 변수 중 어떻게 불확실성이 가장 낮아지는 변수와 그 기준을 찾는가?

앞서 이야기한 불확실성(엔트로피)를 최소화 하는 알고리즘을 통해서 찾음

ex) https://ratsgo.github.io/machine%20learning/2017/03/26/tree/

 

 

 

 

 

 

 

 

 

먼저 설명드릴 지표는 엔트로피(entropy)입니다. m개의 레코드가 속하는 A영역에 대한 엔트로피는 아래와 같은 식으로 정의됩니다. (Pk=A영역에 속하는 레코드 가운데 k 범주에 속하는 레코드의 비율)

Entropy(A)=−∑mk=1pklog2(pk)

 

이 식을 바탕으로 오렌지색 박스로 둘러쌓인 A 영역의 엔트로피를 구해보겠습니다. 전체 16개(m=16) 가운데 빨간색 동그라미(범주=1)는 10개, 파란색(범주=2)은 6개이군요. 그럼 A 영역의 엔트로피는 다음과 같습니다.

Entropy(A)=−1016log2(1016)−616log2(616)≈0.95

 

여기서 A 영역에 빨간색 점선을 그어 두 개 부분집합(R1, R2)으로 분할한다고 가정해 봅시다. 두 개 이상 영역에 대한 엔트로피 공식은 아래 식과 같습니다. 이 공식에 의해 분할 수 A 영역의 엔트로피를 아래와 같이 각각 구할 수 있습니다. (Ri=분할 전 레코드 가운데 분할 후 i 영역에 속하는 레코드의 비율)

그럼 분기 전과 분기 후의 엔트로피가 어떻게 변화했는지 볼까요? 분기 전 엔트로피가 0.95였는데 분할한 뒤에 0.75가 됐군요. 0.2만큼 엔트로피 감소(=불확실성 감소=순도 증가=정보획득)한 걸로 봐서 의사결정나무 모델은 분할한 것이 분할 전보다 낫다는 판단 하에 데이터를 두 개의 부분집합으로 나누게 됩니다. 다시 한번 말씀드리지만 의사결정나무는 구분 뒤 각 영역의 순도(homogeneity)가 증가/불확실성(엔트로피)가 최대한 감소하도록 하는 방향으로 학습을 진행합니다.

Entropy(A)=∑di=1Ri(−∑mk=1pklog2(pk))

 

Entropy(A)=0.5×(−78log2(78)−18log2(18))+0.5×(−38log2(38)−58log2(58))≈0.75

 

  • question 4) 주어진 데이터만 가지고 경계를 계속 만들면 주어진 모델에만 잘 작동하는(=과적합된) 모델이 되지 않는가?

이러한 부분을 방지하기 위해 가지치기(Pruning)을 통해서 적절히 과적합된 가지를 제거

왼쪽의 복잡한 가지를 pruning하여 오른쪽과 같이 적당히 general model을 만듦

 

2. Ensemble

  • question 5) 왜 Random forest가 아니라 Ensemble인가?

앙상블은 다수의 학습 알고리즘을 결합해서 단일 학습 알고리즘에 비해 더 좋은 결과를 얻기 위한 방법으로 Random Forest와 GBM계열의 모델은 모두 Ensemble방식을 사용하며 Bagging 방식과 Boosting 방식으로 나눌 수 있다. 이번에 소개하는 RF와 GBM 모두 Ensemble방식이고 학습 알고리즘을 Decision Tree를 기반으로 하기 때문에 앞서 DT에 대해 간략하게 설명하였다.

 

2-1. Bagging(Bootstrap Aggregating) & Random Forest

  • question 6) Bagging이란 무엇이고 어디에 왜 사용하는 건가?

Bagging은 분산 투자를 하는 것 처럼 다수의 학습 알고리즘을 이용해서 하나의 알고리즘에서 발생할 수 있는 과적합(Overfitting)을 줄여 안정된 모형을 만드는 것이 목적이다.

대표적으로 Random Forest모형에 사용되고 사용자가 사전에 지정한 개수의 Decision Tree의 결과를 지정된 방식(주로 voting을 통해 과반수의 결과 사용)으로 최종 결정한다.

여러 Random sampled Tree의 결과과 모여서 하나의 Forest의 형태를 이룸

2-2. Boosting

Boosting방식은 앞서 Bagging과 마찬가지로 다수의 학습 알고리즘을 사용한다는 공통점이 있다. 하지만 여러 균등한 크기의 Sampled Decision Tree가 아니라 하나의 베이스 트리를 기반으로 다른 트리들을 생성해가면서 최적의 트리를 찾는 다는 점이 다르다. 즉 여러개의 학습 알고리즘을 독립적으로 학습하는것이 아니라 순차적으로 학습시키는 과정.

(Bagging이 시험 공부를 하기 위해 여러 문제집을 푸는 방식이라면 Boosting은 한 문제집에서 틀린 파트를 선정해서 그에 관한 문제집을 사서 풀고, 또 그 문제집에서 틀린 파트에 대한 문제집을 푸는 과정을 반복…)

Boosting 방식에 기반한 모델들 : ADAboost, GBM, XGB, LightGBM, CatBoost 등등…

 

3. Boosting Models

3-1. GBM(Gradient Boosting Model)

위의 그림 GBM의 원리를 나타내는 그림으로 약한 학습기(Weak Learner) D1을 만들고 D1의 약점(Prediction Error)를 보완해서 두 번째 Weak Learner인 D2를 만든다. 이와 같은 과정을 반복하여 만든 Weak Learner를 합치면 하나의 강한 학습기(Strong Learner)가 탄생한다.

  • Question 7) 학습을 한다고 하는데 무엇을 기준으로 학습하는 것이고 어떻게 학습이 되는건지 알 수 있는가?

http://www.ccs.neu.edu/home/vip/teach/MLcourse/4_boosting/slides/gradient_boosting.pdf

https://www.frontiersin.org/articles/10.3389/fnbot.2013.00021/full

손실 함수(Loss function)을 통해 예측 모델의 오류를 정량화한다(앞의 엔트로피 개념과 유사). 이 때 학습은 손실함수를 최소화하는 parameter를 찾는 과정이고 그 방법 중 하나가 Gradient Descent방식이다. 손실함수를 parameter로 미분해서 기울기를 구하고 그 값이 작아지도록 parameter를 변경하면 손실함수가 최소화될 수 있다.

θi+1=θi–ρ∂J∂θi

Gradient Boosting은 이 탐색 과정이 함수 공간에서 이루어진다고 생각하면 쉽다. 그래서 손실함수를 파라미터가 아니라 현재까지 학습된 모델 함수로 미분한다.

θi+1=θi–ρ∂J∂θi

 

파라미터 공간에서는 계산된 기울기를 따라서 학습률(Learning Rate, ρ )에 맞춰 θ 를 업데이트하면 된다. 함수 공간에서는 어떤 일이 생기는지 알아보기 위해 손실함수가 Squared Error:12(y–fi)2 인 경우를 보자. 이때 기울기는 y–fi 로서, 다름아닌 잔차값이다. 모델 함수 (fi) 에 잔차 (y–fi) 를 더하면 당연히 y가 나오겠지만, 이건 의미가 없다.

대신 Gradient Boosting은 이 미분값을 다음 모델(Weak Learner)의 타겟으로 넘긴다. Squared Error를 쓰는 경우를 예로 들면, 현재 모델의 잔차를 타겟으로 놓고 새로운 모델 피팅을 한다. 기존 모델은 이 새로운 모델을 흡수해서 Bias를 줄인다. 그리고 다시 잔차를 구하고 모델을 피팅해서 더하기를 반복한다. 매우 단순하고 직관적인 방법인데, 이걸 손실함수를 L2로 설정한 Gradient Boosting으로 설명할 수 있다.

정리하면 Gradient Boosting에서는 Gradient가 현재까지 학습된 모델의 약점을 드러내는 역할을 하고, 다른 모델이 그걸 중점적으로 보완해서 성능을 Boosting한다. 위에서는 L2 손실함수를 썼지만 미분만 가능하다면 다른 손실함수도 얼마든지 쓸 수 있다는 것이 장점이다. 부스팅 알고리즘의 특성상 계속 약점(오분류/잔차)을 보완하려고 하기 때문에 잘못된 레이블링이나 아웃라이어에 필요 이상으로 민감할 수 있다.

 

3-2. XGboost

https://towardsdatascience.com/boosting-algorithm-xgboost-4d9ec0207d

XGB  : Obj(Θ)=L(θ) + Ω(Θ)

L : loss function, Ω : regularization term

Standard GBM에 없는 Regulariztion term을 사용해서 모델이 과적합 되는 것을 방지할 수 있고, 메모리 최적화를 통해 평균적으로 좀 더 빠른 연산 속도를 보임.

 

 

3-3. LightGBM

https://www.analyticsvidhya.com/blog/2017/06/which-algorithm-takes-the-crown-light-gbm-vs-xgboost/

Light GBM is a fast, distributed, high-performance gradient boosting framework based on decision tree algorithm, used for ranking, classification and many other machine learning tasks.

Since it is based on decision tree algorithms, it splits the tree leaf wise with the best fit whereas other boosting algorithms split the tree depth wise or level wise rather than leaf-wise. So when growing on the same leaf in Light GBM, the leaf-wise algorithm can reduce more loss than the level-wise algorithm and hence results in much better accuracy which can rarely be achieved by any of the existing boosting algorithms. Also, it is surprisingly very fast, hence the word ‘Light’.

Before is a diagrammatic representation by the makers of the Light GBM to explain the difference clearly.

Level-wise tree growth in XGBOOST.

Leaf wise tree growth in Light GBM.

 

Leaf wise splits lead to increase in complexity and may lead to overfitting and it can be overcome by specifying another parameter max-depth which specifies the depth to which splitting will occur.
