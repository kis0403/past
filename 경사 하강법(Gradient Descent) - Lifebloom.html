경사 하강법(Gradient Descent)

많은 머신 러닝 모델들은 오류를 최소화하는 방향으로 문제를 해결하려고 합니다. 이러한 최적화 문제를 풀기위한 방법 중 하나인 경사 하강법에 대해 알아보겠습니다.

 

함수 f를 최대화 시키는 방법 중 하나는 임의의 시작점을 잡은 뒤, 그 gradient를 계산하고 gradient의 방향으로 조금씩 이동하는 과정을 여러번 반복하는 것입니다.

 

f가 단변수 함수의 경우 임의의 점 x에서의 미분 값은 x의 변화에 따른 f(x)의 변화를 뜻합니다.

def difference_quotient(f,x,h):
    return (f(x+h)-f(x))/h

대부분의 경우에서 도함수(derivative)를 구하는 것은 어렵지 않습니다. 하지만 도함수를 구할 수 없다면 h에 아주 작은 값을 대입해서 그 근사값을 구할 수 있습니다.

def partial_difference_quotient(f, v, i, h):
 
    # add h to just the i-th element of v
    w = [v_j + (h if j == i else 0)
         for j, v_j in enumerate(v)]
 
    return (f(w) - f(v)) / h
 
def estimate_gradient(f, v, h=0.00001):
    return [partial_difference_quotient(f, v, i, h)
            for i, _ in enumerate(v)]

 

아래 함수는 sum of squares의 값을 구하는 함수로 v가 0 vector일때 가장 최소가 된다. 이러한 사실을 알 수 없다고 가정했을 때, 경사 하강법을 이용해서 최소값을 구해보겠습니다.

def sum_of_squares(v):
    """computes the sum of squared elements in v"""
    return sum(v_i ** 2 for v_i in v)

 

random 시작점을 잡고, gradient가 매우 작아질 때 까지 경사의 반대 방향으로 조금씩 이동시킵니다.

def step(v, direction, step_size):
    """move step_size in the direction from v"""
    return [v_i + step_size * direction_i
            for v_i, direction_i in zip(v, direction)]
 
def sum_of_squares_gradient(v):
    return [2 * v_i for v_i in v]

    next_v = step(v, gradient, -0.0001)       # take a negative gradient step
    if distance(next_v, v) < tolerance:     # stop if we're converging
        break
    v = next_v                              # continue if we're not

v가 [0,0,0]에 매우 가까워지는 것을 볼 수 있는데 tolerance를 작게 할수록 더 가까이 수렴하는 것을 볼 수 있습니다.

 

위의 식에서 경사도의 음수 만큼 이동하는데 그 과정에서 적절한 step-size가 필요합니다. step-size는 neural net에서 learning rate로 불리기도 하는데 기계적인 방법보다는 modeler의 경험에 의존하는 art에 가까운 방식으로 정해집니다. (물론 데이터 맞춰 학습 속도를 조절하는 Adagrad, RMSprop등 여러 approach가 있습니다.)

이 때 , function에 부적절한 값을 넣어 오류가 발생하는 경우를 방지하기 위해 오류가 발생한 경우 무한대(infinity)를 반환하도록 합니다.

def safe(f):
    """define a new function that wraps f and return it"""
    def safe_f(*args, **kwargs):
        try:
            return f(*args, **kwargs)
        except:
            return float('inf')         # this means "infinity" in Python
    return safe_f

def minimize_batch(target_fn, gradient_fn, theta_0, tolerance=0.000001):
    """use gradient descent to find theta that minimizes target function"""
 
    step_sizes = [100, 10, 1, 0.1, 0.01, 0.001, 0.0001, 0.00001]
 
    theta = theta_0                           # set theta to initial value
    target_fn = safe(target_fn)               # safe version of target_fn
    value = target_fn(theta)                  # value we're minimizing
 
    while True:
        gradient = gradient_fn(theta)
        next_thetas = [step(theta, gradient, -step_size)
                       for step_size in step_sizes]
 
        # choose the one that minimizes the error function
        next_theta = min(next_thetas, key=target_fn)
        next_value = target_fn(next_theta)
 
        # stop if we're "converging"
        if abs(value - next_value) < tolerance:
            return theta
        else:
            theta, value = next_theta, next_value

 

함수는 반복문을 돌면서 데이터 셋 전체를 살펴보기 때문에 minimize_batch라고 하였습니다.

 

이런 경우에는 데이터 전체의 gradient를 반복문마다 계산하기 때문에 계산 시간이 매우 오래 걸립니다. 따라서 데이터 전체의 오류값이 각각 데이터 포인트의 오류값의 합과 같다는 성질을 이용해 데이터 포인트 하나에 대한 gradient를 계산하는 SGD(Stochastic grdient descent)를 사용합니다. SGD는 수렴이 될 때까지 데이터셋을 반복적으로 사용합니다.

def in_random_order(data):
    """generator that returns the elements of data in random order"""
    indexes = [i for i, _ in enumerate(data)]  # create a list of indexes
    random.shuffle(indexes)                    # shuffle them
    for i in indexes:                          # return the data in that order
        yield data[i]

def minimize_stochastic(target_fn, gradient_fn, x, y, theta_0, alpha_0=0.01):
 
    data = list(zip(x, y))
    theta = theta_0                             # initial guess
    alpha = alpha_0                             # initial step size
    min_theta, min_value = None, float("inf")   # the minimum so far
    iterations_with_no_improvement = 0
 
    # if we ever go 100 iterations with no improvement, stop
    while iterations_with_no_improvement < 100:
        value = sum( target_fn(x_i, y_i, theta) for x_i, y_i in data )
 
        if value < min_value:
            # if we've found a new minimum, remember it
            # and go back to the original step size
            min_theta, min_value = theta, value
            iterations_with_no_improvement = 0
            alpha = alpha_0
        else:
            # otherwise we're not improving, so try shrinking the step size
            iterations_with_no_improvement += 1
            alpha *= 0.9
 
        # and take a gradient step for each of the data points
        for x_i, y_i in in_random_order(data):
            gradient_i = gradient_fn(x_i, y_i, theta)
            theta = vector_subtract(theta, scalar_multiply(alpha, gradient_i))
 
    return min_theta
